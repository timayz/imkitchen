### Frontend Architecture Requirements
[Source: architecture/frontend-architecture.md]
- **Component Organization:**
  ```
  templates/
  ├── layouts/
  │   ├── base.html              # PWA shell with meta tags
  │   └── authenticated.html     # Layout for logged-in users
  ├── components/
  │   ├── navigation.html        # Bottom navigation bar
  │   └── loading_states.html    # Skeleton loading screens
  └── fragments/               # TwinSpark response fragments
  ```

- **PWA Template Integration:**
  ```html
  <!-- base.html PWA Enhancement -->
  <head>
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="theme-color" content="#059669">
    <link rel="manifest" href="/static/manifest.json">
    <link rel="apple-touch-icon" href="/static/images/icon-192x192.png">
  </head>
  ```

- **TwinSpark Progressive Enhancement:** All dynamic interactions must use ts-* attributes, no custom JavaScript required for core functionality

### Component Integration Requirements
[Source: architecture/components.md]
- **IMKitchen Web Server Component:** HTTP endpoints returning HTML fragments for TwinSpark ts-target replacement, static asset serving for Tailwind CSS, local TwinSpark JavaScript, and images

### Coding Standards Requirements
[Source: architecture/coding-standards.md]

- **Testing Required:** Minimum 90% test coverage, TDD for all new features
- **Naming Conventions:** Templates use snake_case.html (e.g., `mobile_navigation.html`, `loading_states.html`)
