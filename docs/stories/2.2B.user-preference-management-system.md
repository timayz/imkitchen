# Story 2.2B: User Preference Management System

## Status

Decomposed

## Story

**As a** cooking enthusiast,  
**I want** comprehensive preference settings that customize meal planning to my lifestyle,  
**so that** automated plans respect my cooking constraints and personal favorites.

## Acceptance Criteria

1. User preference settings for maximum prep time per meal and complexity preferences
2. Weekend vs. weekday cooking pattern recognition with appropriate recipe assignment
3. Preference for certain recipes marked as "favorites" with increased rotation frequency
4. Reset rotation option allowing users to restart their recipe cycle
5. Preference validation with sensible defaults and constraint checking
6. Frontend preference configuration screens with intuitive controls

### Technical Architecture Context

**Database Schema Extensions** [Source: docs/architecture/9-database-schema.md]

**Existing User Fields to Leverage**:
- `preferred_meal_complexity` (simple/moderate/complex) 
- `cooking_skill_level` (beginner/intermediate/advanced)
- `max_cook_time` INTEGER for prep time preferences
- `preference_learning_data` JSONB for algorithm data storage
- `rotation_reset_count` for tracking reset history

**New Database Requirements**:
```sql
-- User-Recipe Favorites Junction Table
CREATE TABLE user_recipe_favorites (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    recipe_id UUID REFERENCES recipes(id) ON DELETE CASCADE,
    favorited_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    weight_multiplier DECIMAL(3,2) DEFAULT 1.5, -- Increased rotation frequency
    UNIQUE(user_id, recipe_id)
);

-- Weekly Availability Patterns
CREATE TABLE user_weekly_patterns (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    day_of_week INTEGER CHECK (day_of_week BETWEEN 0 AND 6), -- 0=Sunday
    max_prep_time INTEGER DEFAULT 60, -- minutes
    preferred_complexity VARCHAR(20) DEFAULT 'moderate',
    is_weekend_pattern BOOLEAN DEFAULT false,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

```yaml
  summary: Retrieve current user preferences
  
# Favorites Management

  summary: Mark recipe as favorite
  
DELETE /api/v1/users/favorites/{recipeId}:
  summary: Remove recipe from favorites

  summary: List user's favorite recipes with weighting

# Rotation Control
POST /api/v1/users/rotation/reset:
  summary: Reset user's rotation cycle with confirmation
  requestBody:
    confirmReset: boolean (required)
    
GET /api/v1/users/rotation/stats:
  summary: Get rotation statistics and variety metrics
```
