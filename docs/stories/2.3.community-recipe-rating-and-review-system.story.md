# Story 2.3: Community Recipe Rating and Review System

## Status
🎉 **DONE** - Implementation complete, QA passed, production ready

## Story
**As a** user,
**I want** to rate and review recipes from other community members,
**so that** I can discover high-quality recipes and share my cooking experiences.

## Acceptance Criteria

**ALL criteria must be implemented using TDD + DDD + CQRS + ES + Validation + Askama methodology:**

1. **DDD Rating Domain:** RecipeRating and RecipeReview aggregates with validated Star rating (1-5), ReviewText, Helpfulness value objects using validator derive macros
2. **Input Validation:** Star rating (1-5 integers), review text length (10-2000 chars), photo uploads (optional, max 5MB), rating uniqueness per user-recipe pair
3. **Evento Event Sourcing:** RecipeRated, RecipeReviewed, ReviewEdited, ReviewDeleted, ReviewHelpfulnessVoted events with Evento serialization and audit trail
4. **Evento Commands:** RateRecipeCommand, CreateReviewCommand, EditReviewCommand, VoteHelpfulnessCommand processed by Evento command handlers with validation
5. **Evento Queries:** RatingsByRecipeQuery, ReviewsByRecipeQuery, UserRatingHistoryQuery handled by Evento query handlers with statistical aggregation
6. **Askama + Tailwind Templates:** RatingStars.html, ReviewForm.html, ReviewList.html, RatingDistribution.html with Tailwind styling and type-safe review data binding
7. **TwinSpark Rating Management:** Rating forms with `ts-req="/ratings"` and `ts-target="#rating-section"` → Axum handlers → validate commands → render Askama fragments
8. **JavaScript-Free Review System:** Star selection, review submission, helpfulness voting via TwinSpark attributes without custom JavaScript
9. **Domain Services:** RatingAggregationService, ReviewModerationService, StatisticalWeightingService with validated inputs encapsulate complex business logic
10. **Evento Projections:** RecipeRatingAggregateView, ReviewListView, RatingDistributionView maintained by Evento projection builders with real-time updates
11. **TwinSpark + Tailwind Examples:** Star ratings with `text-yellow-400 hover:text-yellow-500 cursor-pointer` styling and review helpfulness buttons
12. **JavaScript-Free Interactions:** Photo upload previews, review editing modals, rating distribution charts all via TwinSpark HTML attributes and server responses
13. **Evento Real-time Updates:** Rating averages update instantly through Evento event streams with statistical weighting and Askama template re-rendering
14. **Evento Review Moderation:** Automated spam detection events and manual moderation workflow with ReviewModerated, ReviewFlagged events
15. **TDD Review Testing:** Write tests first covering rating submission, review creation, helpfulness voting, and moderation workflows
16. **Evento Notification System:** Review notifications integrated with Evento event streams for recipe creators with privacy controls
17. **Crate Isolation:** Rating and review domain in recipe crate independent of presentation concerns, web crate depends on recipe crate interfaces

### Coding Standards Requirements
[Source: architecture/coding-standards.md]
- **Type Safety Across Boundaries:** All data models must be defined in shared crate with validation
- **Event Sourcing Consistency:** Never mutate aggregates directly - only through Evento create() builder pattern
- **Template Security:** Always use Askama filters for user input, never raw HTML injection
- **TwinSpark Patterns:** All dynamic interactions must use ts-* attributes, no custom JavaScript
- **Error Handling:** All handlers must return Result<T, AppError> with proper error conversion
- **Validation First:** Input validation with validator before any business logic execution
- **Testing Required:** Minimum 90% test coverage, TDD for all new features
- **Crate Boundaries:** Domain crates cannot depend on web crate, only shared types
- **Naming Conventions:** snake_case for templates/handlers, PascalCase for commands/events

### Rating Domain Model Requirements
[Source: architecture/data-models.md (extended for ratings)]
- **RecipeRating Struct:** rating_id: Uuid, user_id: Uuid, recipe_id: Uuid, star_rating: u8 (1-5), created_at: DateTime<Utc>
- **RecipeReview Struct:** review_id: Uuid, rating_id: Uuid, review_text: String (10-2000 chars), photos: Vec<String>, helpfulness_score: i32
- **Rating Constraints:** One rating per user per recipe, star rating validation (1-5), review text length validation
- **Helpfulness System:** Track helpful/unhelpful votes with user_id tracking to prevent duplicate voting
- **Validation:** Use validator derive macros for star rating range, text length, and uniqueness constraints
- **Evento Integration:** Implement AggregatorName, Encode, Decode derives for all rating and review events
- **Moderation Support:** ReviewModerationStatus enum for content filtering and spam prevention

### Database Schema Requirements
[Source: architecture/database-schema.md (extended for ratings)]
- **Recipe Ratings Table:** rating_id TEXT PRIMARY KEY, user_id TEXT NOT NULL, recipe_id TEXT NOT NULL, star_rating INTEGER CHECK(star_rating BETWEEN 1 AND 5)
- **Recipe Reviews Table:** review_id TEXT PRIMARY KEY, rating_id TEXT REFERENCES recipe_ratings(rating_id), review_text TEXT, moderation_status TEXT
- **Review Helpfulness Table:** user_id TEXT, review_id TEXT, is_helpful BOOLEAN, voted_at DATETIME, PRIMARY KEY (user_id, review_id)
- **Rating Aggregates Table:** recipe_id TEXT PRIMARY KEY, average_rating REAL, total_ratings INTEGER, rating_distribution TEXT (JSON)
- **Indexes:** CREATE INDEX idx_ratings_recipe_id, idx_ratings_user_id, idx_reviews_rating_id, idx_helpfulness_review_id
- **Constraints:** UNIQUE(user_id, recipe_id) for recipe_ratings table to prevent duplicate ratings
- **Evento Auto-Management:** Event storage handled automatically by Evento framework

### TwinSpark + Askama Patterns
[Source: architecture/api-specification.md, architecture/frontend-architecture.md]
- **Rating Submission:** `<form ts-req="/recipes/{id}/ratings" ts-target="#rating-section">` for star rating submission

- **Helpfulness Voting:** `<button ts-req="/reviews/{id}/helpful" ts-target="#helpfulness-count" ts-trigger="click">` for voting
- **Live Rating Updates:** `<div ts-req="/recipes/{id}/rating-summary" ts-trigger="load delay:2s">` for real-time aggregation
- **Fragment Updates:** Server returns HTML fragments that replace target elements with updated rating data
- **Template Structure:** Use RatingStarsTemplate, ReviewFormTemplate, ReviewListTemplate with type-safe data binding
- **Styling Integration:** Apply Tailwind classes in Askama templates with star hover states and responsive design
