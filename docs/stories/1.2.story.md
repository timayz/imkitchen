### Data Models

**Core models required for authentication and user management:**

**User Model:**

- id: UUID primary key for security and distribution
- email: unique string for authentication identifier
- name: string for display name in household coordination
- passwordHash: string for secure password storage
- dietaryPreferences: string array for meal planning (vegetarian, vegan, keto, paleo, gluten-free, dairy-free)
- allergies: string array for food safety and recipe filtering
- householdId: UUID foreign key to shared household data
- language: string for interface localization (en, es, fr, de)
- timezone: string for meal planning and cooking schedules
- timestamps: createdAt, updatedAt for audit trails
  [Source: data-models.md#user-model]

**Household Model:**

- id: UUID primary key for shared kitchen spaces
- name: string for household display name
- settings: JSONB for flexible household preferences including measurement units, shared inventory access, meal planning permissions
- members relationship to User model for family/roommate coordination
- timestamps: createdAt, updatedAt for audit trails
  [Source: data-models.md#household-model]

### Database Schema

**PostgreSQL Schema Design:**

### Technical Constraints

**Database Technology:**

- PostgreSQL 15+ for ACID compliance and complex relationships
- Prisma ORM for type-safe database access and migrations
- Connection pooling for production scalability

**TypeScript Integration:**

- Prisma Client generates TypeScript types automatically
- All database operations must be type-safe

**Security Requirements:**

- UUID primary keys to prevent enumeration attacks
