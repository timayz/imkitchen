# Story 1.4: Basic Meal Calendar Display

## Status

Done

## Story

**As a** meal planner,  
**I want** a visual weekly calendar showing my planned meals,  
**so that** I can see my upcoming week at a glance and manually plan meals.

## Acceptance Criteria

1. Weekly calendar view with breakfast, lunch, dinner slots for each day
2. Drag-and-drop recipe assignment to calendar slots from recipe collection
3. Color coding for meal complexity and advance preparation requirements
4. Navigation between weeks with clear date indicators
5. Empty state guidance encouraging users to add recipes and plan meals
6. Mobile-optimized calendar with touch-friendly interactions

### Database Schema Details

[Source: architecture/9-database-schema.md#core-tables]

**Meal Plans Table Schema:**

```sql
CREATE TABLE meal_plans (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    week_start DATE NOT NULL,
    week_end DATE GENERATED ALWAYS AS (week_start + INTERVAL '6 days') STORED,
    meals JSONB NOT NULL, -- {"monday": [{"mealType": "breakfast", "recipeId": "...", "servings": 2}]}
    generation_algorithm_version VARCHAR(20) DEFAULT 'v1.0',
    generation_parameters JSONB,
    generation_duration_ms INTEGER,
    status VARCHAR(20) DEFAULT 'active' CHECK (status IN ('draft', 'active', 'archived', 'deleted')),
    user_feedback JSONB DEFAULT '{}',
    completion_percentage DECIMAL(5,2) DEFAULT 0.0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    archived_at TIMESTAMP WITH TIME ZONE
);
```

**Required Indexes:**

- `idx_meal_plans_user_id` - B-tree index for user isolation
- `idx_meal_plans_week_start` - B-tree index for date-based queries
- `idx_meal_plans_user_week` - Composite index for user + week queries
- `idx_meal_plans_meals` - GIN index for meals JSONB queries
