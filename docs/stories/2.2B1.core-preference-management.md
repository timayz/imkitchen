### Technical Architecture Context

**Database Schema Extensions** [Source: docs/architecture/9-database-schema.md]

**Existing User Fields to Leverage**:

- `preferred_meal_complexity` (simple/moderate/complex)
- `cooking_skill_level` (beginner/intermediate/advanced)
- `max_cook_time` INTEGER for prep time preferences
- `preference_learning_data` JSONB for algorithm data storage

**New Database Requirements**:

```sql
-- User-Recipe Favorites Junction Table (for future stories)
CREATE TABLE user_recipe_favorites (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    recipe_id UUID REFERENCES recipes(id) ON DELETE CASCADE,
    favorited_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    weight_multiplier DECIMAL(3,2) DEFAULT 1.5,
    UNIQUE(user_id, recipe_id)
);

-- Weekly Availability Patterns (for future stories)
CREATE TABLE user_weekly_patterns (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    day_of_week INTEGER CHECK (day_of_week BETWEEN 0 AND 6),
    max_prep_time INTEGER DEFAULT 60,
    preferred_complexity VARCHAR(20) DEFAULT 'moderate',
    is_weekend_pattern BOOLEAN DEFAULT false,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```
