### Data Models

**Core Entities from Architecture:** [Source: architecture/data-models.md]

- **User:** UUID id, email, displayName, dietaryRestrictions, cookingSkillLevel, weeklyAvailability, preferredMealComplexity, rotationResetCount
- **Recipe:** UUID id, title, ingredients array, instructions array, prepTime/cookTime, complexity enum, cuisineType, mealType enum, servings, imageUrl
- **MealPlan:** UUID id, weekStartDate, generationType enum, generatedAt timestamp, totalEstimatedTime, isActive boolean
- **MealPlanEntry:** UUID id, date, mealType, isManualOverride, scheduledPrepTime, isCompleted
- **RecipeRating:** UUID id, rating (1-5), review text, difficulty assessment, wouldCookAgain boolean

### Database Schema

**PostgreSQL Schema Requirements:** [Source: architecture/9-database-schema.md]

- Core tables: users, recipes, meal_plans, recipe_ratings with proper indexes and constraints
- Supporting tables: shopping_lists, user_preference_history
- Performance tables: recipe_performance_metrics with daily aggregation
- Database functions: update_recipe_rating_aggregates(), search_recipes() with full-text search
- Views: active_user_meal_plans, recipe_recommendations with weighted ratings

### API Specifications

**REST API Design:** [Source: architecture/api-specification.md]

- Health endpoints: GET /health, GET /readiness for system monitoring
- Auth endpoints: POST /auth/login, /auth/register, /auth/refresh with JWT tokens
- Protected routes under /api/v1/ with Bearer authentication
- Rate limiting: 5 requests/minute for meal plan generation endpoint
- Response time requirement: <2 seconds for meal plan generation

### Backend Architecture

**Go Service Architecture:** [Source: architecture/11-backend-architecture.md]

- Hexagonal architecture with Domain, Application, Infrastructure layers
- Gin framework with middleware pipeline (Logger, Recovery, CORS, RequestID, Security)
- RedisCacheService with intelligent caching for meal plans and recipe search
- Graceful shutdown with configurable timeouts
- Health check endpoints returning system status

### Frontend Architecture

**Lynx.js Mobile Application:** [Source: architecture/10-frontend-architecture.md]

- MVVM + State Management pattern with Redux Toolkit/Zustand
- Atomic design system (atoms, molecules, organisms, templates)
- React Navigation for cross-platform routing
- API client with caching and offline-first architecture
- Cross-platform builds for iOS/Android/Web
