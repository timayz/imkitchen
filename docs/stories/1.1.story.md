### Tech Stack Configuration
**Backend Framework**: axum 0.8+ with Rust 1.70+ 
**Database**: PostgreSQL 17+ with ACID compliance, JSON support  
**Cache**: Redis 8.2+ for sessions and caching  
**Frontend**: Askama 0.14+ templating with twinspark-js reactivity  
**Build**: Cargo for Rust compilation, Vite 5.x for CSS/JS bundling  
**IaC**: Docker Compose 2.x for development environments  
*[Source: architecture/tech-stack.md#technology-stack-table]*

### Project Structure Requirements
Unified structure with `src/` containing:
- `main.rs`: Application entry point and server setup
- `routes/`: HTTP handlers (pages/ and fragments/ subdirectories)
- `services/`: Business logic services
- `repositories/`: Data access layer
- `models/`: Domain models
- `middleware/`: HTTP middleware (auth, cors, logging)
- `config/`: Configuration management (database.rs, settings.rs)
*[Source: architecture/unified-project-structure.md#project-structure]*

### Database Configuration
**Connection**: PostgreSQL with sqlx for async operations
**Migrations**: Use sqlx migration system
**Schema**: UUID primary keys, JSONB for flexible data
**Access Pattern**: Repository pattern with PgPool
*[Source: architecture/database-schema.md, architecture/backend-architecture.md#data-access-layer]*

### Development Workflow
**Prerequisites**: Rust toolchain, Node.js 20, PostgreSQL 17, Redis
**Setup Commands**: `cargo build`, `npm install`, `sqlx migrate run`
**Environment**: `.env` file with DATABASE_URL, REDIS_URL, JWT_SECRET
**Development**: `docker-compose up -d postgres redis`, `cargo run`
*[Source: architecture/development-workflow.md#local-development-setup]*

### Coding Standards
**Naming**: snake_case for functions/files, PascalCase for structs
**Database**: Never modify existing migrations, use new files
**Environment**: Access config through config module, not `std::env`
**Templates**: Use Askama built-in escaping, no raw HTML
*[Source: architecture/coding-standards.md#critical-fullstack-rules]*

### Testing Framework
**Backend**: tokio-test for async Rust testing
**Integration**: axum-test for API endpoint testing
**Directory**: `tests/` with unit/, integration/, e2e/ subdirectories
**Location**: Tests in `tests/unit/`, `tests/integration/`
*[Source: architecture/testing-strategy.md#test-organization]*
