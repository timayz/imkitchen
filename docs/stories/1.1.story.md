### Technical Constraints

- **Next.js Version**: Must be 14+ for App Router support
- **TypeScript**: Version 5.0+ required for type safety
- **Node.js**: Version 18+ required for Next.js 14 compatibility
- **Package Manager**: Use pnpm for consistent dependency management
- **CSS Framework**: Tailwind CSS 3+ for utility-first styling
- **Testing**: Jest + React Testing Library for unit tests, Playwright for E2E
  [Source: tech-stack.md#technology-stack-table]

### Naming Conventions

- **Files**: kebab-case for pages and API routes
- **Components**: PascalCase for React components
- **Variables**: camelCase for variables and functions
- **Constants**: SCREAMING_SNAKE_CASE for environment variables
- **Types**: PascalCase for TypeScript interfaces
  [Source: coding-standards.md#naming-conventions]

### Critical Fullstack Rules to Implement

- Environment variables must be accessed through config objects in `src/lib/config.ts`
- All shared types must be defined in `src/types/` directory
- Follow unified project structure exactly as specified
- Use proper Git branching and commit conventions
  [Source: coding-standards.md#critical-fullstack-rules]

### Testing

**Test File Locations**:

- Component tests: `tests/components/`
- API tests: `tests/api/`
- E2E tests: `tests/e2e/`

**Test Standards**:

- Use Jest + React Testing Library for frontend unit tests
- Use Jest + Supertest for backend API tests
- Use Playwright for end-to-end testing
- Follow testing pyramid: Many unit tests, some integration tests, few E2E tests

**Testing Frameworks and Patterns**:

- Mock external dependencies using Jest mocks
- Use React Testing Library for component testing with proper accessibility queries
- Test voice interaction components with custom event simulation
- Follow the test examples provided in testing strategy
