# Story 2.1: Basic Inventory Item Management

## Status

Done

## Story

**As a** home cook,
**I want** to add, edit, and remove items from my pantry and refrigerator inventory,
**so that** I can track what ingredients I have available for cooking.

## Acceptance Criteria

1. Inventory page displays separate sections for pantry, refrigerator, and freezer items
2. Add item form includes fields for name, quantity, unit of measurement, category, and expiration date
3. Item editing allows updating all fields with proper validation
4. Delete functionality with confirmation prevents accidental removal
5. Search functionality filters inventory by item name or category
6. Inventory items display with clear visual indicators for quantity and expiration status
7. Form validation ensures required fields and prevents duplicate entries
8. Database models support inventory relationships with users and proper indexing
9. Mobile-optimized interface with swipe-to-edit gestures for efficiency
10. Auto-save functionality prevents data loss during form interactions

### Data Models

**InventoryItem Model** - Core data structure for inventory management:

- id: UUID primary key for unique identification
- name: string - Ingredient name with i18n support
- quantity: number - Current quantity available
- unit: MeasurementUnit - Standardized measurement units
- category: InventoryCategory - Organization categories (proteins, vegetables, fruits, grains, dairy, spices, condiments, beverages, baking, frozen)
- location: StorageLocation - Storage locations (pantry, refrigerator, freezer)
- expirationDate: Date - Expiration tracking for waste prevention
- purchaseDate: Date - Acquisition date
- estimatedCost: number - Optional cost tracking
- householdId: string - Links to shared household data
- addedBy: string - User ID who added the item
  [Source: architecture/data-models.md#inventoryitem-model]

**User and Household Relationships:**

- InventoryItems belong to one Household (shared inventory system)
- Items are added by specific Users (addedBy field)
- Household members can view and manage shared inventory
  [Source: architecture/data-models.md#relationships]

### API Specifications

**Inventory Management Endpoints:**

- GET `/api/inventory` - Retrieve household inventory with optional filtering
  - Query parameters: location, category, expiringSoon
  - Returns array of InventoryItem objects
  - Authentication required with household context
- POST `/api/inventory` - Create new inventory item
  - Request body: InventoryItemCreate schema
  - Returns created InventoryItem with 201 status
  - Validates required fields and prevents duplicates
- PUT `/api/inventory/[itemId]` - Update existing inventory item
  - Request body: InventoryItemUpdate schema (partial updates allowed)
  - Returns updated InventoryItem
  - Household ownership verification required
- DELETE `/api/inventory/[itemId]` - Remove inventory item
  - Returns 204 status on success
  - Household ownership verification required
    [Source: architecture/api-specification.md#rest-api-specification]

**Authentication Integration:**

- All inventory routes require NextAuth.js session validation
- Household context automatically added via middleware
- User context available through session for addedBy tracking
  [Source: architecture/backend-architecture.md#authentication-and-authorization]
