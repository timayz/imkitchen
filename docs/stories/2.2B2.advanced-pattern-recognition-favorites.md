**Pattern Recognition Logic**:

```go
// Weekend vs. Weekday Pattern Recognition
type WeeklyPattern struct {
    DayOfWeek         int     `json:"dayOfWeek"`         // 0=Sunday, 6=Saturday
    MaxPrepTime       int     `json:"maxPrepTime"`       // Minutes
    PreferredComplexity string `json:"preferredComplexity"` // simple/moderate/complex
    IsWeekendPattern  bool    `json:"isWeekendPattern"`
}

// Enhanced rotation logic with day-aware assignment
func (r *RotationService) AssignRecipeForDay(day time.Weekday, userPatterns []WeeklyPattern) (*Recipe, error) {
    pattern := r.getPatternForDay(day, userPatterns)
    recipes := r.filterRecipesByPattern(pattern)
    return r.selectWithFavoritesWeighting(recipes), nil
}
```

**API Specifications** [Source: Story 2.2B1 foundation]

**New API Endpoints**:

```yaml
# Favorites Management
POST /api/v1/users/favorites/{recipeId}:
  summary: Mark recipe as favorite
  responses:
    201: Favorite created successfully
    409: Recipe already favorited

DELETE /api/v1/users/favorites/{recipeId}:
  summary: Remove recipe from favorites
  responses:
    200: Favorite removed successfully
    404: Favorite not found

GET /api/v1/users/favorites:
  summary: List user's favorite recipes with weighting
  parameters:
    - page: integer (pagination)
    - limit: integer (max 50)
  responses:
    200: Array of favorite recipes with metadata

# Pattern Configuration (extends preferences API)
PUT /api/v1/users/preferences/patterns:
  summary: Update weekly cooking patterns
  requestBody:
    weeklyPatterns: array of WeeklyPattern objects
```
